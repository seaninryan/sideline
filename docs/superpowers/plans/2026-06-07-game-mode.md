# Game Mode (Live-Entry Takeover) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A full-screen "game mode" for live match entry — scoreboard + one big-button wizard stage at a time (Team → Event → Player), hiding all other chrome until exited.

**Architecture:** A `gm` state object inside `MatchTracker` (null = off; `{stage, team?, ev?, off?}` = on). When on, the top bar, menu, modal panels, settings, colour picker, and tabs are not rendered; the body renders the wizard instead of a tab. All entry goes through the existing live-entry machinery (`buildEventLine`, `addLive`, `whoGrid`, `completeSub`, `evEnabled`/`phase`, `undoTarget`/`doUndo`) — no parser changes. **Not a fixed overlay** — this codebase learned that fixed overlays don't receive taps in mobile webviews (see CLAUDE.md, Share/Backup panels).

**Tech Stack:** Single-file React 18 + Babel standalone (`index.html` is the source of truth — edit it directly). No build step.

**Spec:** `docs/superpowers/specs/2026-06-07-game-mode-design.md`

**Testing reality:** The repo's only automated tests cover the pure parser/raw-edit helpers (`node tools/run-tests.js`), which this feature doesn't touch — they must simply stay green. UI work is verified by the esbuild JSX syntax check plus manual walkthrough. There is no UI test harness; do not invent one.

**Verification commands (used in every task):**

```bash
# JSX syntax check (needs Node 18+; nvm use 18)
sed -n '/<script type="text\/babel"/,/<\/script>/p' index.html | sed '1d;$d' > /tmp/sideline-app.jsx
npx esbuild /tmp/sideline-app.jsx --loader:.jsx=jsx --outfile=/dev/null

# Parser regression tests
node tools/run-tests.js
```

Expected: esbuild exits 0 with no output; run-tests prints all PASS (canonical sample: Racoons 2-6 v Wildebeests 2-7, Loss, Rick 2-4 (4 frees), Morty 0-1, leadChanges 1, timesLevel 3, maxLead 6, 0 warnings).

### Task 2: The wizard (stages, team-aware addLive, undo row)

**Files:**
- Modify: `index.html`

- [ ] **Step 1: Make `liveLine`/`addLive` team-aware**

The current code (line ~1445) hardcodes `lvTeam`. Add an optional param so game mode can pass its own team without touching the old panel's behaviour:

```jsx
  const liveLine = (ev, player, team = lvTeam) => buildEventLine(ev, team, player, String(new Date().getMinutes()));
```

and in `addLive` (line ~1461):

```jsx
  const addLive = (ev, player, team = lvTeam) => {
    if (!evEnabled(ev)) return; // e.g. FT tapped while a "Who?" pick was pending
    const l = liveLine(ev, player, team);
    if (!l) return;
    append(l);
    setLvEvent(null);
    setSavedMsg(`Added “${l}”`); setTimeout(() => setSavedMsg(""), 1800);
  };
```

(Only the signature and the `liveLine` call change.)

              <button className="mt-add alt" disabled={!undoTarget} onClick={doUndo}>↩ Undo</button>
            </div>
          </div>
        )}
